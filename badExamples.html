Consistency Shortcuts Feedback Dialogue Error Handling Permit Reversal Locus Control Memory Load

Consistency

Failing to maintain consistency can confuse users and make the site feel unprofessional. For example, inconsistent navigation menus or page designs force users to relearn how to interact with each page.

Shortcuts

Lack of shortcuts frustrates power users, making navigation or common actions time-consuming. For instance, a site that doesn't support keyboard shortcuts forces frequent users to rely on repetitive clicks.

Feedback

Insufficient feedback leaves users uncertain about their actions. For instance, a button that doesn’t visually change or provide confirmation after being clicked confuses users.

Dialogue

Poor dialogue design leaves users unsure if their actions were successful. For example, not showing a success message after submitting a form can create doubt.

Error Handling

Without clear error messages, users don’t know what went wrong or how to fix it. For example, a vague “Error occurred” message doesn’t help users resolve the issue.

Permit Reversal

Failure to allow easy reversal of actions makes users afraid of making mistakes. For instance, a site without an undo option for deletions creates frustration.

Locus Control

Users feel disempowered if they lack control. For example, automatic page refreshes or pop-ups disrupt the user experience and make users feel frustrated.

Memory Load

Overloading users’ memory with too much information at once can cause confusion. For instance, a form requiring users to memorize details from previous steps increases cognitive load.
